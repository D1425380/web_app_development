# PRD - 任務管理系統

## 1. 專案概述
- **背景與動機**
  - 現代學生常需要同時管理課業、專案與團隊任務，缺乏有效的工具會導致任務遺忘、進度不清、協作不順。
- **目標用戶**
  - 大學生、研究生及任何需要在學業或課外專案中分配與追蹤任務的學生族群。
- **核心價值主張**
  - 提供直覺式的任務建立與分配介面，結合提醒與進度追蹤功能，提升個人與團隊工作效率。

## 2. 功能需求 (User Stories)
| 功能 | 使用者故事 |
|------|------------|
| 任務建立 | **作為** 學生，**我希望** 能快速輸入任務名稱、描述、截止日期與優先級，**以便** 在開始工作前就能明確掌握任務需求。 |
| 任務分配 | **作為** 團隊領袖，**我希望** 把任務指派給特定成員，並設定分工比例，**以便** 讓每個成員了解自己的責任。 |
| 進度追蹤 | **作為** 成員，**我希望** 在任務卡上更新完成百分比與實際完成情況，**以便** 團隊即時掌握整體進度。 |
| 提醒通知 | **作為** 使用者，**我希望** 在任務即將到期或有變更時收到提醒，**以便** 不會錯過重要期限。 |
| 紀錄與查詢 | **作為** 使用者，**我希望** 能搜尋歷史任務、查看完成紀錄與統計圖表，**以便** 回顧工作成效並做未來規劃。 |
| 任務評論 (額外功能) | **作為** 成員，**我希望** 能在任務下方留下評論與附件，**以便** 促進資訊交流與問題追蹤。 |

## 3. 非功能需求
- **技術棧**: 後端使用 Flask + SQLite，前端使用 HTML、CSS、JavaScript，模板採 Jinja2。
- **效能**: 支援同時 100 位使用者，任務列表載入時間不超過 2 秒。
- **安全考量**: 使用 HTTPS、密碼雜湊儲存、基本的 RBAC（角色：管理者、成員）。
- **可維護性**: 代碼遵循 PEP8，加入單元測試，使用 Git 版控。
- **可擴充性**: 設計資料庫 schema 時保留外鍵與標籤機制，未來能加入多平台同步。

## 4. MVP 範圍
| 層級 | 功能 |
|------|------|
| Must Have | 任務建立、任務分配、進度追蹤、提醒通知、紀錄與查詢 |
| Should Have | 任務評論、任務過期自動標記、簡易儀表板統計 |
| Nice to Have | 手機端響應式設計、第三方日曆同步、AI 任務建議 |

## 5. 專案成員與分工
| 角色 | 責任 |
|------|------|
| 專案負責人 | 需求確認、時程規劃、最終驗收 |
| 前端開發者 | UI/UX 設計、HTML/CSS/JS 實作 |
| 後端開發者 | Flask API、資料庫設計、測試撰寫 |
| 測試工程師 | 撰寫單元與整合測試、執行測試計畫 |
| UI/UX 設計師 | Wireframe、視覺設計、使用者流程圖 |
